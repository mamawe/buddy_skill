# NoFluff

拒绝阅读 AI 水文——评估文章各章节的实际阅读价值，识别 AI 生成的低质量内容（AI slop），输出章节热力图、阅读建议与理由。

基于三圈层检测框架，整合领域写作惯例修正、代码段中性评分、短文章置信度处理、LLM 特征鉴别等改进。

---

```bash
npx skills add mamawe/buddy_skill --skill NoFluff
```

- WorkBuddy
- Claude Code
- Cursor

## 功能说明

### 核心能力

1. **哪些章节值得读** —— 即使一个章节是 AI 生成的，它仍然可能有信息价值
2. **这篇文章整体值不值得花时间** —— 综合章节质量、信息增量，给出最终建议
3. **代码段中性评分** —— 代码是可复制的，不因为"信息密度高"就得高分
4. **LLM 特征鉴别** —— 识别文章是哪个 LLM 生成的（ChatGPT/Claude/DeepSeek 等），给予标注

### 触发词

- `这篇文章是不是水文`
- `评估这篇文章的阅读价值`
- `这篇文章值不值得读`
- `给我一个阅读建议`
- `分析各段落的价值`
- `NoFluff 评估一下`
- `识别 AI 特征`

## 评分维度

### 章节级评分（5 维度）

| 维度 | 权重 | 定义 |
|------|------|------|
| 信息密度 | 30% | 单位字数内包含多少有效信息 |
| 引用质量 | 25% | 文本内部是否提供了可验证的线索 |
| 结构清晰度 | 20% | 章节内部逻辑是否清晰 |
| 独特性 | 15% | 内容是否超越了常识和公开可检索的信息 |
| 与全文的贡献度 | 10% | 这个章节对理解整篇文章的价值有多大 |

### 全文可阅读性评分（4 维度）

| 维度 | 权重 | 定义 |
|------|------|------|
| 净信息收益 | 35% | 读完全文，读者能获得多少真正有价值的信息 |
| 可跳读性 | 25% | 读者能否通过跳读快速获取核心价值 |
| 内容独特性 | 25% | 这篇文章有多少内容是搜索引擎找不到的 |
| 时间性价比 | 15% | 考虑文章长度和价值分布，时间投入是否合理 |

## 阅读建议分级

| 建议 | 分值区间 | 描述 |
|------|---------|------|
| **值得精读** | 80-100 | 有独家经验、非共识观点、写作质量极高 |
| **建议跳读** | 55-79 | 质量不错，但可能有 AI 辅助或有一些水文特征 |
| **仅看摘要** | 30-54 | 整体价值低，只需了解核心结论 |
| **建议跳过** | 0-29 | 水文，阅读价值不足以支付时间成本 |

## 与 truthlens 的关系

- `truthlens`：回答"这篇文章有多少是 AI 写的"
- `NoFluff`：进一步回答"这篇文章值不值得读、怎么读"
- 两个 skill 可以独立使用，也可以串联使用
