# NoFluff

> 拒绝阅读 AI 水文 —— 评估文章各章节的实际阅读价值

## 一句话介绍

AI 写的文章太多，分不清哪些是水文。**NoFluff** 用三圈层检测框架，给文章做「阅读价值体检」，输出章节热力图 + 阅读建议。

## 安装

```bash
npx skills add mamawe/buddy_skill --skill NoFluff
```

适用：WorkBuddy · Claude Code · Cursor · 任何支持 Agent Skills 的 AI

---

## 核心功能

| 功能 | 说明 |
|------|------|
| 🔍 章节热力图 | 每章节标注「值得读 / 可跳过 / 水文警告」 |
| 📊 阅读建议 | 给出具体阅读策略（跳读 / 精读 / 跳过） |
| 🤖 AI slop 检测 | 识别 AI 生成的低质量内容特征 |
| 📝 理由输出 | 每个判断都给出具体理由，不黑盒 |

---

## 三圈层检测框架

```
第一圈层：结构特征（目录 / 标题模式 / 章节密度）
    ↓
第二圈层：内容特征（句式重复 / 空话密度 / 代码段中性评分）
    ↓
第三圈层：领域修正（根据文章领域调整判定标准）
    ↓
输出：章节热力图 + 阅读建议 + 理由
```

## 使用示例

**方式一：直接粘贴文章 URL**
> "用 NoFluff 评估这篇文章：https://..."

**方式二：粘贴文章文本**
> "用 NoFluff 评估下边这篇文章：[粘贴全文]"

**方式三：上传文件**
> "用 NoFluff 评估这个文件" → 上传 PDF / Markdown

---

## 输出示例

```
📊 阅读价值评估报告

第一章：引言  ★★★★☆  建议精读
  → 理由：问题定义清晰，有具体场景描述

第二章：技术原理  ★★☆☆☆  可跳读
  → 理由：大量 AI 生成式空话，「综上所述」「值得注意的是」高频出现

第三章：实战案例  ★★★★★  必读
  → 理由：真实代码片段，有具体错误信息和处理方式
```
