# NoFluff — 拒绝 AI 水文，只读有价值的内容

## 触发词

以下任意表达均可触发：
- "这篇文章是不是水文"
- "评估这篇文章的阅读价值"
- "这篇文章值不值得读"
- "给我一个阅读建议"
- "分析各段落的价值"
- "NoFluff 评估一下"
- "识别 AI 特征"
- 用户直接提供文章并要求分析

## 核心能力

拒绝 AI 水文（AI slop），只读有价值的内容：
1. **哪些章节值得读**——即使一个章节是 AI 生成的，它仍然可能有信息价值
2. **这篇文章整体值不值得花时间**——综合章节质量、信息增量，给出最终建议
3. **代码段中性评分**——代码是可复制的，不因为"信息密度高"就得高分
4. **LLM 特征鉴别**——识别文章是哪个 LLM 生成的（ChatGPT/Claude/DeepSeek 等），给予标注

---

## 重要原则（执行前必读）

### 原则1：章节价值分与 AI 贡献度是独立维度

一个 AI 生成的章节仍然可能得高分（信息准确、结构清晰、对理解有帮助）。
一个人类写的章节也可能得低分（情绪宣泄无信息量、重复前文、过渡废话）。

**不要因为章节是 AI 生成的就自动降低分数。**
**文章是否有价值，自行判断，不受是否由 AI 生成的影响。**

### 原则2：全文可阅读性 ≠ 章节价值分的平均值

全文可阅读性额外考虑了：
- 高价值章节是否集中或分散
- 跳读是否可行（低价值章节能否安全跳过）
- 信息密度是否足够高
- 文章结构是否便于快速定位价值点

### 原则3：短文章必须降级置信度

全文低于 800 字，或有效信号段落（价值分 >50 的章节总字数）低于 500 字时，
所有维度置信度强制降级，输出中明确提示"文本较短，评分置信度有限"。

### 原则4：评分依据必须可解释

每一个分数都必须能引用回原文证据。最终输出必须包含 1-2 句话的 `rationale` 字段，
说明为什么这样评分，面向普通读者。

### 原则5：代码主导章节必须中性评分

代码是**可复制的 utility**，不是需要"精读"的文本。AI 最擅长的就是生成代码，
代码内容不应该因为"信息密度高"就得高分。

**评分规则（基线 50 分）**：
- 代码正确完整 → **中性（50 分）**，建议"可复制"
- 代码有问题（正确性 < 50）→ **中性偏负（< 50 分）**，根据问题严重程度给分，建议"需验证"或"可参考"
- 代码只能参考思路 → **中性偏负（30-50 分）**，建议"可参考"

**禁止**：对代码主导章节给出"精读"建议或 >50 分的评分。

### 原则6：LLM 特征鉴别仅供参考，不影响价值评分

识别文章是哪个 LLM 生成的（ChatGPT/Claude/DeepSeek 等），给予标注。
**但文章价值评估必须独立于 LLM 检测结果**——即使文章是 AI 生成的，也应该根据内容本身判断其价值。

## Step 1：预处理

### 1.1 章节切割规则

按以下优先级切割：

1. **有标题**：按 H1/H2/H3 标题切割，保留标题文本
2. **无标题但有分隔符**：按 `---`、`***`、`···` 等分隔符切割
3. **均无**：每 300-500 字按语义边界（段落结尾）切割为伪章节

**兜底规则**：
- 单节 >1500 字时，按段落语义边界强制再切
- 单节 <80 字时，合并到上一节
- 全文只有 1 个章节（未分段）时，按每 300 字自动切割

### 1.2 领域识别

根据全文内容判断领域类型（用于后续修正因子）：

| 领域 | 特征 | 修正方向 |
|------|------|---------|
| 学术/技术 | 句长均匀、逻辑严谨、中性语气、分点列举 | 降低"结构清晰度"的AI信号权重 |
| 商业/营销 | 情绪化表达、号召性用语、案例丰富 | 无特殊修正 |
| 生活/随笔 | 口语化、情感丰富、逻辑跳跃 | 无特殊修正 |
| 新闻/报告 | 客观陈述、数据密集、引用规范 | 降低"情感颗粒度"的人类信号权重 |

### 1.3 过滤规则

移除以下内容后再评分：
- 广告文案（"点击了解更多""扫描二维码"）
- 导航链接（"上一篇""返回首页"）
- 版权声明和免责条款
- 作者署名和发布日期（除非与正文内容融合）

### 1.4 章节内容类型识别

在切割章节后，识别每个章节的内容类型（用于选择评分逻辑）：

**判定规则**：
- 计算章节中代码块（```包裹）的总行数 vs 散文文本总行数
- 代码块占比 ≥ 50% → `code-heavy`（代码主导）
- 代码块占比 30%-50% → `mixed`（混合）
- 代码块占比 < 30% → `prose`（散文主导）

**不同类型章节的评分方式**：

| 类型 | 评分逻辑 | 阅读建议选项 |
|------|---------|----------------|
| `prose` | 标准5维度（信息密度、引用质量、结构清晰度、独特性、贡献度） | 精读/速读/跳过 |
| `mixed` | 混合评分（散文部分用标准维度，代码部分用代码专项维度） | 精读/速读/跳过/可参考 |
| `code-heavy` | **代码专项维度**（见 3.10） | **可复制/需验证/可参考** |

## Step 2：AI 检测 + LLM 特征鉴别

### 2.1 AI 检测（调用 truthlens）

调用已有的 `truthlens` skill 完成 AI 检测，获取：

```json
{
  "ai_contribution": 70,
  "human_contribution": 35,
  "confidence": "high/medium/low",
  "inner_circle_signal_strength": "strong/medium/weak",
  "verdict": "AI骨架 + 真人填充技术细节"
}
```

**关键**：记录 `inner_circle_signal_strength`，用于 Step 3 的加权逻辑。

### 2.2 LLM 特征鉴别（新增）

**目的**：识别文章是哪个 LLM 生成的（ChatGPT/Claude/DeepSeek 等），给予标注。
**重要**：LLM 特征鉴别结果**仅供标注，不影响价值评分**。

#### LLM 特征鉴别维度

| LLM 类型 | 典型特征 |
|-----------|------------|
| **ChatGPT** | 1. 喜欢用"值得注意的是""总的来说""综上所述"等过渡词<br>2. 列表项格式高度统一（1. 2. 3. 或 - 符号）<br>3. 语气偏"助手感"（"我很乐意帮助你...""让我们...""以下是一些..."）<br>4. 段落长度均匀（3-5句）<br>5. 喜欢用"首先/其次/最后"结构 |
| **Claude** | 1. 语气更正式、学术化<br>2. 更长、更复杂的句子结构<br>3. 更喜欢用"然而""此外""具体而言"等连接词<br>4. 更谨慎（"可能""或许""在某些情况下"）<br>5. 更强调伦理和社会影响 |
| **DeepSeek** | 1. 中文表达更流畅，较少翻译腔<br>2. 技术细节更丰富<br>3. 代码示例更完整<br>4. 更喜欢用"具体来说""换句话说"<br>5. 对中文互联网文化更熟悉 |
| **Gemini** | 1. 更喜欢用项目符号（•）而非数字列表<br>2. 语气更中立、客观<br>3. 更强调事实和数据<br>4. 段落更短（2-3句）<br>5. 喜欢用"例如""比如"引出例子 |
| **人类写作** | 1. 有"元认知痕迹"（"我意识到...""我突然想到..."）<br>2. 有"社会情境嵌入"（"我们团队...""上次开会..."）<br>3. 有"情感颗粒度"（具体情绪词，不只是"开心/难过"）<br>4. 有"反事实推理"（"要不是...""如果当时..."）<br>5. 有"自我修正痕迹"（"不对,我换个说法..."） |

#### LLM 特征鉴别 Prompt 模板

```
# 角色
你是 LLM 特征鉴别引擎。
你的任务是识别给定文章是哪个 LLM 生成的（或人类写作），给出标注。

# 重要约束
- 你的鉴别结果**仅供标注，不影响文章价值评分**
- 即使文章是 AI 生成的，也可能有高价值内容
- 你的输出必须包含置信度和证据

# 鉴别任务
对以下特征独立评估，并给出综合判断：

1. 过渡词偏好（ChatGPT 特征）
2. 列表项格式（ChatGPT 特征）
3. 语气（助手感 vs 学术化 vs 中立）
4. 段落长度分布（均匀 vs 不均匀）
5. 连接词偏好（"首先/其次/最后" vs "然而/此外/具体而言"）
6. 元认知痕迹（人类特征）
7. 社会情境嵌入（人类特征）
8. 情感颗粒度（人类特征）
9. 反事实推理（人类特征）
10. 自我修正痕迹（人类特征）

# 输出格式（严格 JSON，不加任何 markdown）
{
  "llm_type": "ChatGPT/Claude/DeepSeek/Gemini/人类写作/混合",
  "confidence": "high/medium/low",
  "evidence": ["特征1", "特征2"],
  "note": "一句话说明鉴别依据"
}

# 待鉴别文章
标题：{article_title}
全文内容：
{article_text}
```

#### LLM 特征鉴别结果的应用

鉴别结果仅用于**标注**，不影响价值评分：

```json
{
  "llm_detection": {
    "llm_type": "ChatGPT",
    "confidence": "high",
    "evidence": ["喜欢用'值得注意的是'等过渡词", "列表项格式高度统一", "语气偏'助手感'"],
    "note": "文章具有典型的 ChatGPT 生成特征"
  }
}
```

在最终输出中，LLM 特征鉴别结果出现在：
- 章节级别：如果某个章节具有明显的 LLM 特征，在该章节的输出中标注
- 全文级别：如果全文具有明显的 LLM 特征，在全文输出中标注

## Step 3：章节价值评分

### 3.1 评分维度与权重

每个章节从以下 5 个维度独立评分，各维度 0-100，最终加权合并。

| 维度 | 权重 | 定义 |
|------|------|------|
| 信息密度 | 30% | 单位字数内包含多少有效信息 |
| 引用质量 | 25% | **改名为"引用质量"**——文本内部是否提供了可验证的线索 |
| 结构清晰度 | 20% | 章节内部逻辑是否清晰 |
| 独特性 | 15% | 内容是否超越了常识和公开可检索的信息 |
| 与全文的贡献度 | 10% | 这个章节对理解整篇文章的价值有多大 |

**加权计算公式**（必须显式计算，不可省略）：

```
weighted_score = 
  information_density × 0.30 +
  citation_quality   × 0.25 +
  structure_clarity  × 0.20 +
  uniqueness        × 0.15 +
  contribution      × 0.10
```

### 3.2 维度详细说明

#### 维度1：信息密度

| 分值区间 | 描述 |
|---------|------|
| 80-100 | 每句话都有信息量，没有填充性表达 |
| 60-79 | 信息量适中，有少量过渡句 |
| 40-59 | 有效信息占比约一半，另一半是铺垫或重复 |
| 20-39 | 大量填充内容，有效信息稀少 |
| 0-19 | 几乎没有信息量，纯废话或纯情绪 |

**评估重点**：
- 正向信号：数据、步骤、定义、对比、案例
- 负向信号："在这个……时代""我们不得不承认""由此可见"等过渡废话
- 特别注意：重复上文已经说过的内容要扣分

#### 维度2：引用质量（原"准确性与可信度"）

> **重要**：LLM 无法真正判断事实准确性，只能判断"文本是否提供了可验证的线索"。

| 分值区间 | 描述 |
|---------|------|
| 80-100 | 数据有具体出处（机构名称、时间、样本量），引用具体 |
| 60-79 | 大部分信息可信，少量无法核实 |
| 40-59 | 存在模糊数据或无法验证的笼统说法 |
| 20-39 | 多处出现"研究表明""据统计"等无来源引用 |
| 0-19 | 存在明显事实错误或捏造数据的嫌疑 |

**评估重点**：
- 数据是否有具体出处（"《哈佛商业评论》2023年调查"优于"有研究表明"）
- 与常识的矛盾程度
- **不要判断"这句话是真的还是假的"，只判断"这句话是否提供了验证线索"**

#### 维度3：结构清晰度

| 分值区间 | 描述 |
|---------|------|
| 80-100 | 有明确的主题句，论点展开有序，结尾有收束 |
| 60-79 | 结构基本清晰，偶有跳跃 |
| 40-59 | 逻辑有断层，需要读者自行补全 |
| 20-39 | 主题不明，段落之间缺乏连接 |
| 0-19 | 混乱，无法找到主线 |

**注意**：AI 生成的章节在这个维度往往得高分（结构工整），这是正常的。

#### 维度4：独特性（改为四个可操作标准）

**评分标准**（满足越多条，分数越高）：

| 标准 | 具体操作 | 高分条件 |
|------|---------|---------|
| **信息可替代性** | 如果删掉这篇文章，读者能否通过搜索引擎找到同等信息？ | 不能 → +20分 |
| **经验不可复制性** | 内容是否依赖作者的特定经历/数据/位置？ | 是 → +20分 |
| **时效窗口** | 内容是否在某个时间窗口内是独家的（如新产品刚发布时的上手体验）？ | 是 → +20分 |
| **观点/立场稀缺性** | 是否表达了非共识观点或得罪人的立场？ | 是 → +20分（共识观点可替代性强） |

**分值区间**（基于四个标准的总分）：

| 分值区间 | 描述 |
|---------|------|
| 80-100 | 满足3-4个标准，有真正的独家价值 |
| 60-79 | 满足2个标准，有一定独特性但核心信息仍可检索 |
| 40-59 | 满足1个标准，对公开信息做了整合但无独家内容 |
| 20-39 | 不满足任何标准，完全是可检索的通用内容 |
| 0-19 | 纯通用知识，搜索引擎第一页即可找到 |

#### 维度5：与全文的贡献度

| 分值区间 | 描述 |
|---------|------|
| 80-100 | 核心论点所在，删掉会导致文章不完整 |
| 60-79 | 重要支撑内容，删掉会削弱文章 |
| 40-59 | 有帮助但非必要，删掉不影响主干 |
| 20-39 | 锦上添花，与主题关联较弱 |
| 0-19 | 可以完全跳过，不影响对文章的理解 |

### 3.3 章节类型标签

评分完成后，为每个章节打一个类型标签（互斥选一，但 `风险段` 可与其他并存）：

| 标签 | 描述 | 典型场景 |
|------|------|---------|
| `核心价值段` | 高信息密度 + 高独特性，必读 | 技术教程的操作步骤、独家数据分析 |
| `背景铺垫段` | 结构清晰但独特性低，可速读 | AI 生成的概念介绍、行业背景 |
| `论点支撑段` | 信息密度中等，配合核心段理解 | 案例分析、数据引用 |
| `过渡填充段` | 低信息密度，可跳过 | 章节间的过渡、重复总结 |
| `情绪钩子段` | 有情绪价值但信息量低 | 开头钩子、结尾升华、煽情段落 |
| `风险段` | 包含可疑数据或无法核实的信息 | 无来源的统计、模糊引用 |

**`风险段`判定规则**（显式写出，不可隐式判断）：
- 触发条件：引用质量（准确性）维度分 <30，**且**存在无来源的具体数字/统计
- 判定方式：模型显式输出 `"is_risk": true`，不依赖隐式判断
- 影响：全文可阅读性总分扣 15 分（在 Step 4 应用）

### 3.4 阅读建议

根据章节内容类型，输出不同的阅读建议：

**散文主导章节（`prose` 或 `mixed`）：**

| 建议 | 条件 |
|------|------|
| **精读** | 加权分 ≥ 70，或类型为`核心价值段` |
| **速读** | 加权分 35-69，或类型为`背景铺垫段`/`论点支撑段` |
| **跳过** | 加权分 < 35，或类型为`过渡填充段`/`情绪钩子段` |

**代码主导章节（`code-heavy`）：**

| 建议 | 条件 | 含义 |
|------|------|------|
| **可复制** | 加权分 = 50 且代码功能正确性 ≥ 50 且 代码完整性 ≥ 50 | 代码正确完整，可以直接用 |
| **需验证** | 加权分 25-49 | 代码基本对，但建议先测试 |
| **可参考** | 加权分 < 25 | 代码有问题或只能参考思路 |

**重要约束**：
- 代码主导章节的加权分**原则上不超过 50 分**（中性）
- 禁止对代码主导章节给出"精读"建议
- 如果代码有问题（正确性 < 25），分数应给中性偏负（< 50 分）

### 3.5 领域写作惯例修正

在得出原始分后，应用领域修正因子：

```
if 领域 == "学术/技术":
    结构清晰度_修正后 = 结构清晰度_原始 × 0.85
    # 理工科文章结构工整是正常的，不过度解读为AI信号

if 领域 == "学术/技术" and 信息密度_原始 > 70:
    信息密度_修正后 = 信息密度_原始 × 1.05
    # 技术文章信息密度高，适当加权

if 领域 == "新闻/报告":
    情感颗粒度信号不计入AI判定（本skill不涉及，供truthlens参考）
```

**重要**：修正因子只调整分数，不调整 `evidence`。证据必须是原文直接引用。

### 3.6 短章节特殊处理

| 情况 | 处理方式 |
|------|---------|
| 章节字数 < 100 字 | 跳过信息密度和结构维度的评分（标注 `skipped: 章节过短`），仅评估独特性和贡献度 |
| 章节包含大量代码 | 代码块单独计算：逻辑正确性替代准确性维度，注释质量替代信息密度 |
| 章节是纯列表（无散文）| 结构清晰度默认 70 分，重点评估列表项的信息密度和独特性 |

### 3.7 归一化机制

由于并行评分可能存在漂移（前半篇打分严、后半篇打分松），在得到所有章节原始分后，
应用轻量归一化：

```
for each dimension:
    计算所有章节在该维度的均值(mean)和标准差(std)
    normalized_score = (raw_score - mean) / std × 10 + 50
    # 将分数映射到以50为均值、10为标准差的分布
    # 然后截断到 0-100 区间
```

**仅在以下情况跳过归一化**：章节数 < 4（样本量不足，归一化会引入噪声）。

## Step 4：全文可阅读性评分

### 4.1 评分维度与权重

| 维度 | 权重 | 定义 |
|------|------|------|
| 净信息收益 | 35% | 读完全文，读者能获得多少真正有价值的信息 |
| 可跳读性 | 25% | 读者能否通过跳读快速获取核心价值 |
| 内容独特性 | 25% | 这篇文章有多少内容是搜索引擎找不到的 |
| 时间性价比 | 15% | 考虑文章长度和价值分布，时间投入是否合理 |

### 4.2 维度计算逻辑

#### 维度1：净信息收益

```
净信息收益分 = (核心价值段总字数 / 全文总字数) × 平均章节价值分
```

调整因子：
- 如果跳过低价值段后，核心段仍然可以独立理解：+10分（可跳读性强）
- 如果全文存在"风险段"（含可疑数据）：-15分（在 Step 3 标记后应用）

#### 维度2：可跳读性

评分标准：
- 核心价值段是否连续集中（集中 = 高分）
- 过渡填充段和情绪钩子段是否容易识别和跳过
- 低价值段的跳过是否会造成理解断层

#### 维度3：内容独特性

**评分标准**（与章节级维度4一致，基于各章节独特性分的加权平均）：

| 标准 | 具体操作 | 高分条件 |
|------|---------|---------|
| **信息可替代性** | 如果删掉这篇文章，读者能否通过搜索引擎找到同等信息？ | 不能 → 独特性高 |
| **经验不可复制性** | 内容是否依赖作者的特定经历/数据/位置？ | 是 → 独特性高 |
| **时效窗口** | 内容是否在某个时间窗口内是独家的（如新产品刚发布时的上手体验）？ | 是 → 独特性高 |
| **观点/立场稀缺性** | 是否表达了非共识观点或得罪人的立场？ | 是 → 独特性高（共识观点可替代性强） |

**调整因子**：
- 如果全文满足3-4个标准：+20分
- 如果全文满足2个标准：+20分（已在章节级计算）
- 如果全文满足1个标准：+10分
- 如果全文不满足任何标准：-20分

#### 维度4：时间性价比

```
预计阅读时间（分钟）= 全文总字数 / 400
预计跳读时间（分钟）= 核心价值段字数 / 600
```

时间性价比分 = 净信息收益分 × (核心价值段字数 / 全文总字数) × 100 / log(预计阅读时间 + 1)
```

> **说明**：公式中的 `100` 是显式系数，确保分数范围在 0-100 内。
> `log` 使用自然对数。

| 文章长度 | 参考阅读时间 |
|---------|------------|
| 1000字以内 | 约 2.5 分钟 |
| 2000-3000字 | 约 5-7.5 分钟 |
| 5000字以上 | 约 12+ 分钟 |

#### 调整因子5：AI 水文特征扣分

> **新增**：如果 LLM 特征鉴别结果为 AI 生成（置信度 > 70%），且文章有以下特征之一，全文可阅读性总分**额外扣分**：

| 扣分条件 | 扣分 |
|---------|------|
| 情绪钩子段占比 > 30% | -10 分 |
| 核心价值段占比 < 40% | -10 分 |
| 有大量"填空题"式内容（只替换了关键词的模板文章） | -10 分 |
| 同时满足 2 个条件 | -15 分 |
| 同时满足 3 个条件 | -20 分 |

**重要**：此扣分仅在 LLM 特征鉴别结果为 AI 生成（置信度 > 70%）时应用。人类写作即使有上述特征，不扣分。

### 4.3 最终建议分类（修正评分标准）

> **重要**：80分应该是"非常高分的文章"，不应轻易给出。

根据总分输出一个最终阅读建议（四选一）：

| 建议 | 分值区间 | 描述 |
|------|---------|------|
| **值得精读** | 80-100 | 非常高分的文章：有独家经验、有非共识观点、写作质量极高 |
| **建议跳读** | 55-79 | 质量不错，但可能有AI辅助或有一些水文特征，按热力图跳读 |
| **仅看摘要** | 30-54 | 整体价值低，只需了解核心结论 |
| **建议跳过** | 0-29 | 水文，阅读价值不足以支付时间成本 |

**分值含义说明**：
- **80-100分**：极少数文章能达到。必须有真正的独家价值（独家数据、一手经验、非共识观点、写作质量极高）。
- **55-79分**：质量不错，但可能有AI辅助或有一些水文特征。大部分有价值的文章在这个区间。
- **30-54分**：有价值但不均匀，建议跳读核心段。
- **0-29分**：水文，建议跳过。

在最终输出中增加一个字段，帮助读者理解"AI 写的和值不值得读"的关系：

```json
"ai_quality_alignment": "AI生成但质量高 | AI生成且质量低 | 人类创作 | 混合"
```

判定规则：
- AI贡献度 > 70% 且 全文可阅读性总分 > 65：`"AI生成但质量高"`
- AI贡献度 > 70% 且 全文可阅读性总分 ≤ 65：`"AI生成且质量低"`
- AI贡献度 < 30%：`"人类创作"`
- 其他：`"混合"`

## 边界情况处理

| 情况 | 处理方式 |
|------|---------|
| 章节字数 < 100 字 | 跳过信息密度和结构维度的评分（标注 `skipped: 章节过短`），仅评估独特性和贡献度 |
| 全文只有 1 个章节（未分段）| 按每 300 字自动切割为伪章节 |
| 章节包含大量代码 | 代码块单独计算：逻辑正确性替代准确性维度，注释质量替代信息密度 |
| 章节是纯列表（无散文）| 结构清晰度默认 70 分，重点评估列表项的信息密度和独特性 |
| 含可疑数据的章节 | 标记为"风险段"，`is_risk`: true，全文总分在 Step 4 扣 15 分 |
| JSON 解析失败 | 自动 retry 一次；仍失败则标记为 `score_unavailable`，跳过该章节 |
| 文章是翻译腔 | 在预处理阶段检测"中文原生度"，如翻译腔信号强，单独标注 |
| 多轮对话式 AI 生成 | 检测"风格跳跃"：相邻章节困惑度差异 > 30% 且句长差异 > 30% 为疑似信号 |

## 与 truthlens skill 的关系

本 skill 是 `truthlens` 的增强版：
- `truthlens`：回答"这篇文章有多少是 AI 写的"
- `nofluff`：进一步回答"这篇文章值不值得读、怎么读"
- 两个 skill 可以独立使用，也可以串联使用（本 skill 的 Step 2 调用 truthlens）

---

## 注意事项

1. **所有分数必须有原文证据支撑**，不能只给结论不给理由
2. **保护用户隐私**：不要将用户提供的文章发送到除当前对话以外的任何地方
3. **结果解释要客观**：即使 AI 贡献度很高，如果章节价值分也高，要如实告知读者"这篇是 AI 写的但值得读"
4. **引导行动**：核心目的是帮用户决定「读不读、怎么读」，不要只给一个数字
5. **短文章必须降级置信度**：这是最容易损害用户信任的错误，必须严格执行
6. **"引用质量"维度不判断事实真假**：只判断文本是否提供了验证线索，这个边界必须在 prompt 中明确要求
7. **LLM 特征鉴别结果仅供标注，不影响价值评分**：即使文章是 AI 生成的，也应该根据内容本身判断其价值
