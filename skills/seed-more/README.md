# seed-more

> 复刻得到大脑「发芽」功能 —— 把知识文档变成会发芽的报告

## 一句话介绍

不是压缩，是**发散**。从文档中提取「种子」，跨域关联你的知识库，生成 Aha 金句报告。

## 安装

```bash
npx skills add mamawe/buddy_skill --skill seed-more
```

适用：WorkBuddy · Claude Code · Cursor · 任何支持 Agent Skills 的 AI

---

## 四步流水线

| 步骤 | 功能 | 说明 |
|------|------|------|
| Step 1 | 提取种子 | 识别 5-10 颗有理论延伸潜力的知识点 |
| Step 2 | 跨域关联 | **优先搜你的 ima 知识库**，搜不到才补互联网 |
| Step 3 | Aha 提炼 | 用悖论式金句引爆张力点 |
| Step 4 | 金句回响 | 原文金句 + 跨域思想回响 |

## 与得到大脑 original 的差异

| 维度 | 得到大脑「发芽」 | seed-more |
|------|----------------|----------|
| 知识来源 | 得到知识库（固定） | **你的** ima 知识库（个性化）|
| 输出格式 | 固定报告 | Markdown，可自定义 |
| 跨域范围 | 预设知识图谱 | 取决于你的知识库 + 互联网 |
| 可复用性 | 一次性 | Skill，可反复使用 |
| 透明度 | 黑盒 | 四步流水线完全透明 |

---

## 为什么每个人的发芽报告都不同？

因为 Step 2 **优先关联你自己的知识库**。

你的知识库决定了你的芽往哪长 —— 这才是真正的个性化。

---

## 使用示例

**方式一：处理 ima 知识库文档**
> "帮我给这篇文档发芽" → 选择知识库 → 选择文档 → 得到发芽报告

**方式二：直接粘贴文本**
> "帮我为下边这段话发芽：[粘贴文本]" → 直接得到发芽报告
